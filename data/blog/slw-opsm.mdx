# SLW - OPSM

조직의 성공을 위해 최적의 동료를 탐색하고, 팀원 간 시너지를 극대화할 수 있도록 돕는 **AI 기반 팀원 매칭 및 팀 핏(Team Fit) 분석 플랫폼** 개발 프로젝트입니다. 고도화된 매칭 알고리즘과 직관적인 대시보드를 구축하여 최적의 팀 빌딩 솔루션을 제공하는 Next.js 기반 웹 플랫폼입니다.

---

## 🛠️ 기술 스택 (Spec)

- **Framework**: Next.js `15` (App Router 기반 프리미엄 아키텍처) + React `19`
- **Language**: TypeScript
- **State Management**: Zustand (도메인별 스토어 설계: `auth`, `teammate`, `search`, `profile`)
- **Data Fetching**: React Query (TanStack Query)
- **Styling**: Tailwind CSS + Shadcn UI (Radix UI) + SCSS + PostCSS
- **Deployment & CI/CD**: Vercel (자동 배포), Bitbucket Pipelines
- **Code Quality**: ESLint + Prettier + Husky & Commitlint (엄격한 커밋 컨벤션 제어)
- **배포 웹사이트**: [slw-opsm.vercel.app](https://slw-opsm.vercel.app/)

---

## 🚀 주요 역할 및 수행 내용

### 1. 차세대 React 19 및 Next.js 15 App Router 도입

- **도메인 기반 폴더 구조 설계**: 전역 공통 컴포넌트(`shared`), 도메인별 기능 컴포넌트(`features`), `shadcn/ui` 기반 하위 컴포넌트(`ui`)로 폴더 구조를 체계화하여 모듈성 및 협업 생산성을 극대화했습니다.
- Next.js 15의 App Router 아키텍처를 도입하여 레이아웃 및 페이지 단위의 서버 사이드 렌더링 최적화를 탑재하고, React 19의 비동기 상태 핸들링 컨셉을 적용하였습니다.

### 2. 다차원 정밀 매칭 및 팀 적합도(Team Fit) 분석 알고리즘 구현

- **이상적인 팀원 설정 및 매칭**: 프로젝트 요구 역량, 역할군, 소프트 스킬 등을 결합하여 '이상적인 팀원 상'을 정의하는 인터페이스와 API를 설계했습니다.
- **다단계 검색 알고리즘**: 데이터베이스 기반의 정밀 매칭(`detail`)과 대략적인 후보군 탐색(`rough`), 그리고 유연하게 피드백을 적용하는 실시간 재검색(`search_again`) 파이프라인을 구축했습니다.
- **시너지 리포트 시각화**: 멤버 간의 화학적·기술적 결합도를 시각적 지표로 분석하는 팀 핏 진단 화면과 고유 ID별 분석 리포트를 구현하여 강점 및 보완점을 사전 예측할 수 있게 지원했습니다.

### 3. Zustand 및 React Query 기반의 상태/데이터 제어 흐름 설계

- **도메인별 분할 Zustand 스토어**: 인증(`auth`), 팀원(`teammate`), 매칭 및 검색(`search`), 내 정보(`profile`) 등으로 전역 상태 저장소를 엄격하게 격리 설계하여 상태 복잡도를 효과적으로 제어했습니다.
- **서버 상태 동기화**: React Query를 도입하여 매칭 요청과 리포트 조회 시 발생하는 서버 상태를 간결하게 캐싱하고 최신 상태로 조율했습니다.

### 4. 대용량 유저 관리 및 보안 검증 기능 개발

- **대량 데이터 일괄 업로드**: 개별 팀원 등록 이외에도 표준 CSV 템플릿(`teammate_template.csv`)을 활용한 대규모 조직 정보 일괄 업로드 기능을 개발했습니다.
- **안전한 인증 체계**: 회원가입 시 신뢰성을 보장하기 위한 이메일 검증 코드 발송 및 인증 로직(`verify_mail`, `check_mail_verification_code`)을 온보딩하고 세션 파괴(회원탈퇴, 로그아웃)와 비밀번호 재설정 작업을 세심히 처리했습니다.

### 5. 엄격한 개발 규칙 적용 및 배포 자동화

- **CI/CD 및 커밋 품질 강화**: Bitbucket Pipelines를 활용하여 변경 내용이 안전하게 검증되도록 파이프라인을 구축했으며, **Husky & Commitlint** 설정을 통해 협업을 위한 커밋 컨벤션을 강제하여 코드베이스의 안정성을 대폭 향상했습니다.
