# Vite 7 → Vite 8 마이그레이션

campaign-ui 프로젝트를 **Vite 7**에서 **Vite 8**으로 업그레이드한 과정과, `master` 브랜치 대비 빌드 성능·산출물 비교 결과를 정리합니다.

## 개요

| 항목                 | Vite 7 (master)                | Vite 8 (마이그레이션)   |
| -------------------- | ------------------------------ | ----------------------- |
| Vite                 | `^7.3.1`                       | `8` (실제 설치: 8.0.16) |
| `@vitejs/plugin-vue` | `^6.0.2`                       | `^6.0.7`                |
| `@tailwindcss/vite`  | `^4.2.1`                       | `^4.3.1`                |
| 번들러               | esbuild (dev) + Rollup (build) | Rolldown + Oxc          |
| 청크 분리            | `manualChunks` (함수)          | `codeSplitting.groups`  |
| Node.js              | 22.14.0                        | 22.14.0 (변경 없음)     |

Vite 8의 핵심 변화는 **esbuild + Rollup 이중 파이프라인을 Rolldown(Rust) + Oxc로 통합**한 것입니다. 대부분의 기존 설정은 호환 레이어로 동작하지만, 장기적으로 `rolldownOptions`·`codeSplitting`·`oxc` 설정으로 이전하는 것이 권장됩니다.

## 마이그레이션 절차

### Phase 0 — 사전 점검

1. `yarn dev`, `yarn build`, `yarn preview` 정상 동작 확인
2. Node.js 20.19+ / 22.12+ 요구사항 충족 확인 (본 프로젝트: 22.14.0)
3. `@vitejs/plugin-vue`, `@tailwindcss/vite` Vite 8 호환 버전 확인
4. `chore/vite8-migration` 브랜치 생성

### Phase 1 — Rolldown 선행 검증 (권장)

공식 권장 경로는 Vite 7 환경에서 `rolldown-vite` 패키지로 번들러만 먼저 교체해 이슈를 분리하는 것입니다.

```json
"vite": "npm:rolldown-vite@7.2.2"
```

패키지 교체 설치 후 다른 조치없이 개발서버 구동에는 성공하였습니다.

![rolldown-vite@7.2.2](/static/images/vite8-migration/vite-1.png)

그러나 `vite.config.ts` 파일을 보면 esbuild 에 legalComments 라는 속성이 없다는 에러가 빌드 실패를 유발할 수 있기에 해당 속성을 제거했습니다.
기존에 정의했던 부분도 꼭 필요하지 않았기 때문입니다. [esbuild.legalComments](https://esbuild.github.io/api/#legal-comments)
![rolldown-vite@7.2.2_esbuild](/static/images/vite8-migration/vite-2.png)

### Phase 2 — Vite 8 본 업그레이드

1. **의존성 업데이트**

   ```bash
   yarn add -D vite@8 @vitejs/plugin-vue@latest
   yarn add -D @tailwindcss/vite@latest
   yarn install
   ```

   ![rolldownOptions](/static/images/vite8-migration/vite-4.png)

2. **설정 마이그레이션**
   - `build.rollupOptions` → `build.rolldownOptions`
     ![rolldownOptions](/static/images/vite8-migration/vite-3.png)
   - `manualChunks` (함수) → `codeSplitting.groups` (regex + priority)
     ![rolldown-vite@7.2.2_esbuild](/static/images/vite8-migration/vite-5.png)
   - `esbuild` 옵션 제거 - Rolldown 선행 검증시 이미 제거함 (Vite 8 기본: Oxc transform + Lightning CSS minify)

3. **검증**
   - `yarn build` — 청크 분리·빌드 시간
   - `yarn dev` — HMR, API proxy (`/campaign-service/admin/`)
   - 주요 화면 스모크 테스트 (Element Plus, FullCalendar, vee-validate/zod)

## 설정 변경 상세

### package.json

```diff
  "devDependencies": {
-   "@vitejs/plugin-vue": "^6.0.2",
+   "@vitejs/plugin-vue": "^6.0.7",
-   "vite": "^7.3.1",
+   "vite": "8",
  }
```

```diff
  "dependencies": {
-   "@tailwindcss/vite": "^4.2.1",
+   "@tailwindcss/vite": "^4.3.1",
  }
```

### vite.config.ts — Before (Vite 7)

```ts
export default defineConfig({
  esbuild: {
    legalComments: 'none',
  },
  build: {
    reportCompressedSize: false,
    chunkSizeWarningLimit: 1200,
    rollupOptions: {
      output: {
        manualChunks(id) {
          if (!id.includes('node_modules')) return
          if (id.includes('@fullcalendar')) return 'fullcalendar'
          if (id.includes('@element-plus/icons-vue')) return 'element-icons'
          if (id.includes('element-plus')) return 'element-plus'
          if (id.includes('vue-router')) return 'vue-router'
          if (id.includes('pinia')) return 'pinia'
          if (id.includes('vee-validate') || id.includes('/zod')) return 'forms'
          if (id.includes('axios')) return 'axios'
          if (id.includes('dayjs')) return 'dayjs'
          if (id.includes('pino')) return 'pino'
          return 'vendor'
        },
      },
    },
  },
  // ...
})
```

### vite.config.ts — After (Vite 8)

```ts
export default defineConfig({
  build: {
    reportCompressedSize: false,
    chunkSizeWarningLimit: 1200,
    rolldownOptions: {
      output: {
        codeSplitting: {
          groups: [
            { name: 'fullcalendar', test: /node_modules[\\/]@fullcalendar/, priority: 100 },
            {
              name: 'element-icons',
              test: /node_modules[\\/]@element-plus[\\/]icons-vue/,
              priority: 90,
            },
            { name: 'element-plus', test: /node_modules[\\/]element-plus[\\/]/, priority: 80 },
            { name: 'vue-router', test: /node_modules[\\/]vue-router[\\/]/, priority: 70 },
            { name: 'pinia', test: /node_modules[\\/]pinia[\\/]/, priority: 60 },
            {
              name: 'forms',
              test: /node_modules[\\/](vee-validate|@vee-validate|zod)[\\/]/,
              priority: 50,
            },
            { name: 'axios', test: /node_modules[\\/]axios[\\/]/, priority: 40 },
            { name: 'dayjs', test: /node_modules[\\/]dayjs[\\/]/, priority: 30 },
            { name: 'pino', test: /node_modules[\\/]pino[\\/]/, priority: 20 },
            { name: 'vendor', test: /node_modules/, priority: 10 },
          ],
        },
      },
    },
  },
  // ...
})
```

### manualChunks → codeSplitting 매핑

기존 `if/else` 순서를 **`priority` 값**으로 표현합니다. 숫자가 클수록 먼저 매칭됩니다.

| 청크            | test 패턴                              | priority |
| --------------- | -------------------------------------- | -------- |
| `fullcalendar`  | `@fullcalendar`                        | 100      |
| `element-icons` | `@element-plus/icons-vue`              | 90       |
| `element-plus`  | `element-plus/` (icons 제외)           | 80       |
| `vue-router`    | `vue-router/`                          | 70       |
| `pinia`         | `pinia/`                               | 60       |
| `forms`         | `vee-validate`, `@vee-validate`, `zod` | 50       |
| `axios`         | `axios/`                               | 40       |
| `dayjs`         | `dayjs/`                               | 30       |
| `pino`          | `pino/`                                | 20       |
| `vendor`        | 나머지 `node_modules`                  | 10       |

> 경로 구분자는 Rolldown 권장대로 `[\\/]`를 사용해 Windows/macOS/Linux를 모두 대응합니다.

## 비교 측정

### 로컬 (Windows)

#### 측정 환경

| 항목      | 값                                                                |
| --------- | ----------------------------------------------------------------- |
| OS        | Windows 11                                                        |
| Node.js   | 22.14.0                                                           |
| Vite 7    | 7.3.1                                                             |
| Vite 8    | 8.0.16                                                            |
| 측정 방법 | git worktree + `yarn install` 후 `dist` 삭제 → `yarn build` × 5회 |
| 공통 설정 | `reportCompressedSize: false`                                     |

#### 빌드 시간

| 지표       | Vite 7.3.1 | Vite 8.0.16 | 변화           |
| ---------- | ---------- | ----------- | -------------- |
| **중앙값** | **7.29s**  | **1.99s**   | **72.7% 단축** |
| 평균       | 7.71s      | 4.29s\*     | 44.4% 단축     |

### Jenkins (배포 환경)

실제 CI/CD 파이프라인에서도 빌드 시간 개선을 확인했습니다. Vite 7·8 각각 **2회** `yarn build`를 실행했으며, 아래는 **2회차** 결과입니다.

| 지표      | Vite 7.3.1 | Vite 8.0.16 | 변화           |
| --------- | ---------- | ----------- | -------------- |
| **2회차** | **8.89s**  | **2.01s**   | **77.4% 단축** |

로컬 중앙값(7.29s → 1.99s)과 Jenkins 2회차(8.89s → 2.01s) 모두 **약 73~77% 수준의 빌드 시간 단축**을 보였습니다. 배포 환경에서도 Vite 8 전환 효과가 동일하게 나타났습니다.

### 산출물 크기

| 항목      | Vite 7              | Vite 8              | 변화                    |
| --------- | ------------------- | ------------------- | ----------------------- |
| JS 합계   | 1,731 KB (34 files) | 1,725 KB (34 files) | -0.3%                   |
| CSS 합계  | 388 KB (15 files)   | 387 KB (15 files)   | -0.3%                   |
| dist 전체 | ~18.9 MB            | ~18.9 MB            | 동일 (이미지·폰트 포함) |

번들 총량 기준 **기능·용량 회귀 없음**으로 판단합니다.

### Vendor 청크 비교

| 청크             | Vite 7 (manualChunks) | Vite 8 (codeSplitting) | 비고                            |
| ---------------- | --------------------- | ---------------------- | ------------------------------- |
| element-plus     | 719.7 KB              | 841.6 KB               | Rolldown 의존성 포함 방식 차이  |
| element-icons    | 166.8 KB              | 166.7 KB               | 동일                            |
| fullcalendar     | 205.5 KB              | 285.0 KB               | 의존성 포함 범위 확대           |
| forms            | 74.7 KB               | 93.2 KB                | zod 등 포함 범위 차이           |
| vue-router       | 24.9 KB               | 24.2 KB                | 동일                            |
| axios            | 36.2 KB               | 35.4 KB                | 동일                            |
| pinia            | 3.7 KB                | 6.0 KB                 | 소폭 증가                       |
| dayjs            | 16.8 KB               | 1.3 KB                 | 본체만 분리, deps는 다른 청크로 |
| pino             | 7.4 KB                | 8.2 KB                 | 동일 수준                       |
| vendor (vue 등)  | 203.8 KB              | —                      | V8에서 분산 배치                |
| rolldown-runtime | —                     | 0.7 KB                 | V8 전용                         |

청크 **이름·분리 전략은 유지**되나, Rolldown `codeSplitting`은 의존성을 청크에 더 많이 포함하는 경향이 있어 **개별 파일 크기 분포는 달라질 수 있습니다**. JS 총합이 거의 동일하므로 런타임 로드 총량 관점에서는 동등합니다.

## 결론

```
Vite 7.3.1 → Vite 8.0.16 마이그레이션 결과 (campaign-ui)

✅ 프로덕션 빌드 (로컬): 7.3s → 2.0s (중앙값 기준 73% 단축)

⚠️  개별 청크 바이트 분포: Rolldown 특성상 일부 청크 크기 재배분 (총량 동일)
```

## 측정 재현 방법

```powershell
# PowerShell — 빌드 시간 5회 측정
1..5 | ForEach-Object {
  if (Test-Path dist) { Remove-Item -Recurse -Force dist }
  Measure-Command { yarn build 2>$null } | Select-Object TotalSeconds
}

# dist JS/CSS 합계
$js = Get-ChildItem dist/assets -Filter *.js
$css = Get-ChildItem dist/assets -Filter *.css
"JS: $([math]::Round(($js | Measure-Object Length -Sum).Sum / 1KB, 1)) KB"
"CSS: $([math]::Round(($css | Measure-Object Length -Sum).Sum / 1KB, 1)) KB"
```

공정한 비교를 위해 동일 Node 버전, `reportCompressedSize: false`, 클린 `yarn install` 환경에서 측정하는 것을 권장합니다.
